# ELK MCP 工具使用指引

## 你需要知道的三个核心概念

在使用 ELK 工具之前，先理解这三个概念的区别，能帮助你更精确地提问：

| 概念 | 说明 | 举例 |
|------|------|------|
| **Kibana 空间（Space）** | Kibana 中的权限隔离单元，不同团队/业务可以有不同空间 | `jijinslb`、`default` |
| **数据视图（Data View）** | Kibana 中定义的索引匹配模式，决定了你能在 Discover 里看到哪些索引 | `jinjinslb_all-*`、`order-service-*` |
| **ES 索引（Index）** | Elasticsearch 中实际存储数据的单元，通常按天/月分片 | `jinjinslb_all-2026.04.27`、`order-service-2026.04.27` |

**它们的关系：**

```
Kibana 空间（Space）
  └── 数据视图（Data View / Index Pattern）
        └── 匹配多个 ES 索引（按天分片）
              └── 索引中包含具体的文档（日志）
```

> 本工具集直接查询 ES 索引，不经过 Kibana 数据视图。但当你提到视图名时，工具会自动用通配符匹配对应的 ES 索引。

## 如何提出好问题

### 信息越完整，查询越精准

**推荐的提问方式：**

> 帮我查一下 `jinjinslb_all` 视图里，今天上午 10 点到 11 点的 ERROR 日志

这个问题包含了三个关键要素：
1. **数据范围** — `jinjinslb_all` 视图
2. **时间范围** — 今天上午 10 点到 11 点
3. **查询条件** — ERROR 日志

**更多好问题示例：**

| 问题 | 包含的信息 |
|------|-----------|
| `order-service 索引最近 1 小时有多少 ERROR？` | 索引名 + 时间 + 条件 |
| `jinjinslb_all 视图里 serviceName 有哪些值？` | 视图名 + 字段探索 |
| `查 order-service 今天 request_time 最慢的 Top 10` | 索引名 + 时间 + 排序需求 |
| `对比昨天和今天 ERROR 数量的趋势` | 时间对比 + 聚合需求 |

### 模糊问题会导致多次尝试

**不推荐的提问方式：**

| 模糊问题 | 问题所在 | 改进方式 |
|---------|---------|---------|
| `帮我查日志` | 缺少索引、时间、条件 | 说明查哪个索引/视图、什么时间段、什么条件 |
| `查 ERROR` | 不知道查哪个索引 | 补充索引名或视图名 |
| `最近有没有异常？` | "异常"定义不明确 | 说明是 ERROR 日志、还是响应慢、还是某个字段异常 |

## 工具调用流程

当你提出查询请求后，AI 会按以下顺序操作：

```
1. es_indices    → 确认索引是否存在
2. es_mapping    → 查看字段结构（确认时间字段名、可用字段）
3. es_count      → 在同一时间范围做无业务条件基线确认（字段不确定或首次查询为空时）
4. es_field_values / es_search 小样本 → 确认候选字段和值格式
5. es_search     → 搜索具体日志
   或 es_aggregate → 统计分析（Top N、趋势、分布等）
```

如果你发现 AI 反复调用工具或报错，通常是因为：
- 索引名不正确 — 可以先问「帮我列一下有哪些索引」
- 字段名或条件值猜错了 — 正确动作是在原时间范围内隔离条件、换字段或用 `es_field_values` 看候选字段值，不应该直接扩大到全天

## 常用场景速查

| 你想做什么 | 怎么问 |
|-----------|-------|
| 看有哪些索引 | `列出所有包含 order 的索引`；如果确实要浏览全部索引，也可以直接说 `列出所有索引` |
| 看索引有哪些字段 | `看一下 order-service-2026.04.27 的字段结构` |
| 搜索 ERROR 日志 | `查 xxx 视图今天的 ERROR 日志` |
| 看某字段有哪些值 | `xxx 索引的 serviceName 字段有哪些值？` |
| 统计 ERROR 数量趋势 | `xxx 视图最近 6 小时 ERROR 按小时分布` |
| 找最慢的请求 | `xxx 索引今天 request_time 最大的 Top 10` |
| 按条件过滤 | `xxx 视图中 serviceName 是 "user-api" 的 ERROR 日志` |

## 注意事项

1. **工具调用必须有时间范围** — 所有搜索和聚合查询都要求带时间范围，最大跨度 7 天（默认配置）。如果你没指定，AI 会按场景默认窗口查询或先追问。
2. **说"视图"还是"索引"都行** — 工具能自动识别，但明确说明会更快。
3. **字段名区分大小写** — 如果不确定字段名，先让 AI 查 mapping。
4. **keyword vs text** — 精确匹配（如 `level = "ERROR"`）用 keyword 字段，全文搜索用 text 字段。大部分字符串字段同时有两个版本：`fieldName`（text）和 `fieldName.keyword`（keyword）。
5. **不要把条件问题当成时间问题** — 如果你问“过去半小时某条件是否正常”，AI 应先固定这半小时排查字段、取值和匹配方式；只有同一时间范围的基线 count 也是 0，才考虑询问是否扩大时间范围。
