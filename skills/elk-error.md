# userId 异常日志查询

根据 userId 查询用户相关的错误、警告和异常日志。全局索引、时间、字段和空结果规则以 MCP instructions 为准。

## 参数

- userId：必须；未提供时先询问。
- 索引范围：来自用户给出的服务、索引、数据视图或业务系统名；缺失时先追问。
- 时间范围：未指定时默认最近 24 小时。
- 服务名：可选，用于缩小索引或 query。

## 流程

1. 用 `es_indices` 确认索引范围；按天切分的日志查询使用通配模式。
2. 用 `es_mapping` 查找 userId、时间、服务名、日志级别、异常/堆栈和消息字段。
3. 字段或 userId 值格式不确定时，在同一时间范围内用 `es_field_values` 或小样本 `es_search` 确认。
4. 调用 `es_search` 查询 userId 相关 ERROR/WARN，按时间倒序，默认 size=30。
5. 结果为空时按全局规则做基线 count，并隔离 userId、level、服务名等条件。

## 输出

- 概要：错误数量、异常类型数量、最近一次出现时间。
- 错误列表：完整日期时间（含时区）｜服务｜错误类型｜消息摘要。
- 如有堆栈，提取关键异常类和位置；数据不足时说明还需要补充哪类查询。
