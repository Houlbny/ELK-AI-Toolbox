# 通用日志搜索

把自由描述转换为一次可复查的 ES 日志检索。全局索引、时间、字段和空结果规则以 MCP instructions 为准。

## 参数

- 关键词或过滤条件：必须，如服务名、错误信息、用户 ID、接口路径、状态码等。
- 索引范围：优先来自用户给出的索引、通配模式、数据视图或业务系统名；用户只是想浏览索引时可以用 `es_indices(pattern="*")`。
- 时间范围：未指定时默认最近 1 小时，查询工具传 `relative_minutes=60`。
- 返回条数：默认 20，按时间倒序。

如果用户只说“查下日志”且没有关键词或索引线索，先追问查哪个索引/视图、什么内容和什么时间。

## 流程

1. 用 `es_indices` 确认索引范围；按天切分的日志查询使用通配模式。
2. 用 `es_mapping` 确认时间字段、消息字段、级别字段和业务条件字段。
3. 根据字段类型构造 query：精确条件用 `term`，文本搜索用 `match`/`match_phrase`，多条件用 `bool`。
4. 字段或取值不确定时，在同一时间范围内用 `es_field_values` 或小样本 `es_search` 确认。
5. 调用 `es_search` 返回明细；结果为空时按全局规则做基线 count 和条件隔离。

## 输出

- 说明查询时间跨度和命中数量。
- 每条日志显示完整日期时间（含时区）、级别、关键字段和消息摘要。
- 结果过多时建议增加过滤条件；结果为空时说明已排查到的字段/条件问题。
