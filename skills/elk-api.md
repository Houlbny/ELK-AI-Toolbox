# 接口错误率分析

按接口路径统计请求量、错误量、错误率和可选趋势。全局索引、时间、字段和空结果规则以 MCP instructions 为准。

## 参数

- 索引范围：来自用户给出的服务、索引、数据视图或业务系统名；缺失时先追问。
- 时间范围：未指定时默认最近 1 小时，查询工具传 `relative_minutes=60`。
- API 路径：可选；未指定则分析所有接口。
- 服务名：可选。

## 流程

1. 用 `es_indices` 确认索引范围；按天切分的日志查询使用通配模式。
2. 用 `es_mapping` 查找时间、API 路径、HTTP 状态码、服务名、耗时字段。
3. 字段或状态码格式不确定时，在同一时间范围内用 `es_field_values` 或小样本 `es_search` 确认。
4. 调用 `es_aggregate`：按 API 路径 terms 聚合，嵌套错误 filter（如 status >= 400 或 level=ERROR），必要时按状态码分布。
5. 用户要求趋势时，追加 date_histogram 聚合；结果为空时按全局规则做基线 count 和条件隔离。

## 输出

| 接口 | 总请求 | 错误数 | 错误率 | 主要错误码 |
|------|-------|-------|-------|----------|

- 按错误率或错误数倒序，并标出高风险接口。
- 趋势描述必须使用完整日期时间（含时区）。
