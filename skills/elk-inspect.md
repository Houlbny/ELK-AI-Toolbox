# 发布后巡检

对比发布前后的错误量和错误类型，判断新版本是否引入异常。全局索引、时间、字段和空结果规则以 MCP instructions 为准。

## 参数

- 服务名：必须；未提供时先询问。
- 发布时间：必须；用户说“刚发布”时调用 `es_server_time` 取当前时间。
- 对比窗口：默认发布前后各 30 分钟。
- 索引范围：来自服务名、索引或数据视图；缺失时先追问。

用户要求单窗口“最近 N 分钟巡检”时，查询工具传 `relative_minutes=N`；发布前后对比窗口按发布时间计算绝对时间。

## 流程

1. 用 `es_indices` 确认索引范围；按天切分的日志查询使用通配模式。
2. 用 `es_mapping` 查找时间、日志级别、服务名、异常类型/错误消息字段。
3. 分别对发布前窗口和发布后窗口调用 `es_count` 统计 ERROR/WARN 数量。
4. 分别调用 `es_aggregate` 统计错误类型 terms 聚合。
5. 对比错误量变化、新增错误类型、消失错误类型和激增错误。
6. 任一窗口查空时，固定该窗口做字段确认和基线 count，不直接改变巡检窗口。

## 输出

- 巡检结论：正常 / 需要关注 / 高风险。
- 服务、发布时间、发布前后窗口都用完整日期时间（含时区）。
- 列出错误量变化、新增错误类型、激增错误和建议动作。
